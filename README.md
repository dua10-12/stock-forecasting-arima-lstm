# stock-forecasting-arima-lstm
Сравнительный анализ моделей ARIMA и LSTM для прогнозирования временных рядов стоимости акций (на примере AAPL)

Этот проект представляет собой анализ и прогнозирование временного ряда цен закрытия акций Apple Inc. (AAPL). Основная цель - сравнить эффективность классической статистической модели **ARIMA** и модели глубокого обучения **LSTM** в задаче прогнозирования финансовых временных рядов.

## Описание проекта
Работа состоит из нескольких ключевых этапов, от загрузки и первичного анализа данных до построения и оценки прогностических моделей. Основное внимание уделяется выявлению закономерностей, подготовке стационарного ряда и сравнению качества прогнозов двух принципиально разных подходов.

## 1. Загрузка и первичный анализ данных

*   **Источник данных:** Yahoo Finance (`yfinance`).
*   **Период:** 01.01.2022 – 01.01.2025.
*   **Анализируемый показатель:** Цена закрытия (`Close`).

**Основные результаты:**
*   Загружено 753 наблюдения.
*   Построены графики цены закрытия и объемов торгов.
*   Проведен описательный статистический анализ, который показал наличие тренда и рост дисперсии к концу периода.

## 2. Декомпозиция и анализ структуры ряда (STL)

Для понимания внутренней структуры ряда была применена декомпозиция STL (Seasonal-Trend decomposition using LOESS) с периодом сезонности 63 дня (≈ один торговый квартал).

**Ключевые выводы:**
*   Исходный ряд нестационарен (тесты ADF и KPSS подтверждают наличие тренда).
*   Декомпозиция выделила ярко выраженный **тренд** и **сезонную компоненту**.
*   Анализ остатков после декомпозиции показал, что они близки к стационарным, но не являются "белым шумом" (не проходят тест на нормальность).
*   Проверка дисперсии (тест Левена) и анализ амплитуды сезонности показали, что дисперсия и амплитуда сезонных колебаний непостоянны во времени. Это указывает на то, что ряд может быть **мультипликативным**, а не аддитивным.

### Преобразование данных (Логарифмирование)

Для стабилизации дисперсии и преобразования возможной мультипликативной модели в аддитивную, был применен **логарифм** к исходному ряду.

*   После логарифмирования сезонная компонента стала более стабильной по амплитуде.
*   Остатки после STL-декомпозиции логарифмированного ряда стали ближе к стационарным (хотя все еще не являются полностью нормально распределенными).
*   **Ключевой метрикой для LSTM стал логарифмированный ряд.**

## 3. Предобработка и анализ стационарности

*   **Тесты:** ADF (Dickey-Fuller) и KPSS.
*   **Вывод:** Исходный ряд (`Close_AAPL`) и его логарифм (`Log_Close`) являются **нестационарными**, что требует дифференцирования для модели ARIMA.
*   **Выбросы:** Для проверки устойчивости данных были применены два метода обнаружения выбросов (локальный тест Ирвина и адаптивный Z-score). Результаты показали, что данные содержат выбросы, что характерно для финансовых временных рядов.

## 4. Моделирование

Для прогнозирования использовались последние 20% данных (≈150 наблюдений).

### Модель ARIMA

*   **Подход:** Классический статистический метод.
*   **Подбор параметров:**
    1.  Автоматический подбор с помощью `pmdarima.auto_arima`.
    2.  Ручной анализ графиков автокорреляции (ACF) и частной автокорреляции (PACF) для ряда первых разностей (`d=1`).
*   **Результат:** Наилучшей была признана модель `ARIMA(0,1,1)`.
*   **Прогноз:** Модель обучена на логарифмированных данных для учета мультипликативности. Полученный прогноз был преобразован обратно в исходный масштаб цен с помощью экспоненты.

### Модель LSTM

*   **Подход:** Нейронная сеть, способная обучаться на долгосрочных зависимостях.
*   **Архитектура:** Последовательная модель (`Sequential`) из двух слоев LSTM и одного полносвязного слоя (`Dense`).
*   **Подготовка данных:**
    *   Использовался тот же логарифмированный ряд (`Log_Close`).
    *   Данные масштабировались с помощью `MinMaxScaler` в диапазон [0, 1] (для LSTM).
    *   Созданы обучающие последовательности (look-back) для предсказания следующего значения.
*   **Прогноз:** Как и для ARIMA, прогноз LSTM был выполнен для логарифмированного ряда и затем преобразован обратно.

## 5. Сравнение результатов

Для оценки точности прогнозов использовались метрики:
*   **MAE (Mean Absolute Error):** Средняя абсолютная ошибка.
*   **RMSE (Root Mean Square Error):** Квадратный корень из среднеквадратичной ошибки.
*   **MAPE (Mean Absolute Percentage Error):** Средняя абсолютная процентная ошибка.

*   **ARIMA:**
    *   MAE: ~ 6.02
    *   RMSE: ~ 6.95
    *   MAPE: ~ 2.70%
    *   **Вердикт:** Очень достойный результат для простой модели. Хорошо улавливает общий тренд, но немного запаздывает на поворотных точках.

*   **LSTM:**
    *   MAE: ~ 10.31
    *   RMSE: ~ 12.37
    *   MAPE: ~ 4.56%
    *   **Вердикт:** Модель оказалась переобученной. Она отлично "запоминает" обучающую выборку, но на тестовой показывает результаты хуже, чем ARIMA. Это частая проблема LSTM на относительно коротких и волатильных временных рядах.

## 6. Выводы

1.  **Победитель:** В данном конкретном случае модель **ARIMA** значительно превзошла LSTM по всем метрикам качества на тестовой выборке.
2.  **Причины успеха ARIMA:** ARIMA отлично справляется с улавливанием линейного тренда и сезонности. Она проще, требует меньше данных и менее склонна к переобучению, чем LSTM на коротких рядах.
3.  **Причины неудачи LSTM:** LSTM — гораздо более сложная модель, которая требует значительно большего объема данных для эффективного обучения, чем было доступно (3 года). Это привело к переобучению (модель запомнила данные, но не смогла научиться обобщать).
4.  **Практическая значимость:** Полученные результаты подчеркивают важность правила *"начинай с простого"*. Перед применением сложных моделей глубокого обучения всегда следует оценивать эффективность классических статистических методов. Для прогнозирования на относительно коротких временных рядах ARIMA часто оказывается не только более простым, но и более эффективным инструментом.
5.  **Направления для улучшения LSTM:** Для повышения качества LSTM потребуется либо значительно более длинная история данных (например, 10+ лет), либо использование более продвинутых методов регуляризации для борьбы с переобучением.
